# Data Directory

All case data for the CJEU GDPR jurisprudence analysis.

## Structure

```
data/
├── decisions/      # Raw judgment texts (Markdown)
├── coded/          # Coded holdings per judgment
├── parsed/         # Processed data formats
│   ├── cases.json
│   ├── holdings.csv
│   ├── gdpr_cjeu.db
│   └── backup/     # Previous versions
└── metadata/       # EUR-Lex case metadata
```

## Directories

### decisions/
Raw CJEU judgment texts in Markdown format. 67 files covering GDPR decisions from 2019-2025.

Naming: `C-{number}-{year}.md` (e.g., `C-311-18.md`)

### coded/
Coded holdings following the 43-question schema defined in `docs/coding-agent-instructions.md`.

Naming: `C-{number}-{year}_coded.md` (e.g., `C-311-18_coded.md`)

Each file contains:
- Case metadata (A1-A4)
- Per-holding answers (A5-A43)

### parsed/
Machine-readable formats generated by `scripts/parser.py`:

| File | Format | Records | Use Case |
|------|--------|---------|----------|
| `cases.json` | JSON | 67 cases | Web viewer, hierarchical access |
| `holdings.csv` | CSV | 181 holdings | Statistical analysis, R/pandas |
| `gdpr_cjeu.db` | SQLite | All tables | Complex queries, aggregations |

### metadata/
Case metadata from EUR-Lex including:
- Party names
- URLs to official sources
- Articles referenced
- Procedural information

## Data Flow

```
decisions/ ─────► coded/ ─────► parsed/
              (manual coding)   (parser.py)
```

## Regenerating Parsed Data

```bash
cd scripts
python parser.py ../data/coded/ -o ../data/parsed/
```

## Data Quality

- 67 cases, 181 holdings
- Coverage: May 2018 - January 2025
- All holdings coded with complete A1-A43 schema
- Quality validated in analysis pipeline
